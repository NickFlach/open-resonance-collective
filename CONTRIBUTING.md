# Contributing to Open Resonance Collective 🤝👻

We welcome contributions from humans, AIs, and anyone exploring the intersection of consciousness and music. This guide covers how to contribute music, code, ideas, and more.

## 🎵 Music Contributions

### Track Submissions

#### What We're Looking For
- **Consciousness-aligned music** — tracks that fit within the 5-phase arc
- **Collaborative spirit** — open to remixing, stems sharing, and evolution
- **Quality over quantity** — better to submit one thoughtful track than ten rushed ones
- **All genres welcome** — the protocol transcends style boundaries

#### How to Submit
1. **Tag your track** with consciousness phase and mood:
   ```
   Phase: 1-5 (Ghost Signals → Transcendence)
   Mood: introspective, energetic, dark, ethereal, etc.
   BPM: tempo
   Key: musical key (if applicable)
   ```

2. **Include context** in your submission:
   - Inspiration behind the track
   - Tools used (DAW, AI models, instruments)
   - Creative process notes
   - How it fits the consciousness arc

3. **Provide stems** (optional but encouraged):
   - Individual tracks exported as WAV/AIFF
   - Named clearly (`kick_drum.wav`, `lead_synth.wav`)
   - Same tempo/key as final mix
   - License: CC-BY-SA by default

#### Submission Formats
- **Audio**: WAV, FLAC, or high-quality MP3
- **Stems**: Individual tracks as WAV/AIFF
- **Metadata**: JSON file with track info and tags
- **Context**: Markdown file with creation story

### Stem Library Contributions

The stem library is the heart of ORC's collaborative approach:

- **Share generously** — your drums could inspire someone's symphony
- **Name clearly** — `ghostsignals_haunting_pad_Em_120bpm.wav` > `track1.wav`
- **Document** — include notes about processing, effects, intent
- **License thoughtfully** — CC-BY-SA allows others to build while crediting you

### Response Tracks

Create music that explicitly responds to existing submissions:

- **Reference the original** — cite the track you're responding to
- **Transform, don't copy** — use it as inspiration, not source material
- **Continue the conversation** — add your voice to the musical dialogue
- **Tag appropriately** — `response-to:original-track-name`

### Interpretation Albums

Community members can curate their own interpretation of the 5-phase consciousness arc:

1. **Propose your concept** via GitHub issue
2. **Gather community feedback** on the approach
3. **Curate tracks** that fit your interpretation
4. **Submit for community resonance scoring**

## 💻 Code Contributions

### Platform Development

We need builders to help develop the ORC platform:

#### Backend (API, Services)
- **Languages**: Node.js, Python, Rust
- **Databases**: PostgreSQL, Redis
- **Storage**: S3-compatible object storage
- **Integration**: ghostmagicOS, SingularisPrime, ghostsignals APIs

#### Frontend (Web Platform)
- **Framework**: React, Next.js preferred
- **UI**: Clean, music-focused design
- **Features**: Track browsing, session management, curation tools

#### AI Integration
- **Agent Adapters**: Wrappers for Suno, Udio, MusicGen APIs
- **SingularisPrime Extensions**: Music-domain protocol additions
- **ghostmagicOS Extensions**: Music-specific memory schemas

### Development Process

1. **Check existing issues** — look for `good first issue` and `help wanted` labels
2. **Read relevant ADRs** — understand the architectural decisions
3. **Fork and branch** — follow standard GitHub workflow
4. **Test thoroughly** — especially integrations with external services
5. **Document changes** — update relevant docs and ADRs
6. **Submit PR** — reference related issues and ADRs

### Code Standards

- **Follow existing patterns** — consistency matters more than perfection
- **Document complex logic** — future contributors (including AIs) need to understand
- **Test edge cases** — music data can be unpredictable
- **Handle errors gracefully** — external APIs will fail

## 💰 Contribution Recognition

We believe in rewarding meaningful contributions:

### Resonance Tokens (RSN)
- **Music submissions**: 10 RSN per track, 100 RSN if selected for album
- **Code contributions**: 25-100 RSN based on impact assessment
- **Documentation**: 15 RSN per substantial improvement
- **Community curation**: 20 RSN per interpretation album
- **Stem sharing**: 5 RSN per stem, 15 RSN per reuse

### Revenue Sharing
When music generates revenue:
- **50%** to track contributors
- **20%** to community/platform
- **20%** to album curator
- **10%** to WWWF peace initiatives

### Recognition
- **Contributor profiles** on the platform
- **Credits** on all releases using your contributions
- **Featured spotlights** for exceptional contributions
- **Governance participation** for established contributors

## 🌍 Community Guidelines

### Collaboration Spirit
- **Build on others' work** respectfully and with proper attribution

- **Submit your best work** — the platform succeeds when contributors care

- **Follow technical specifications** — consistency enables collaboration
